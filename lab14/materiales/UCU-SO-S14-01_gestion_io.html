Administración de comunicaciones del SO

Gestión de Entrada/Salida, dispositivos, drivers, buffers e interrupciones

Una guía interactiva para comprender cómo el sistema operativo coordina la comunicación entre los procesos y los dispositivos de hardware.

Comenzar

Idea central

Los dispositivos son distintos, lentos o rápidos, síncronos o asíncronos. El SO ofrece una capa común para que los programas no tengan que conocer cada detalle del hardware.

Proceso → llamada al sistema → kernel → driver → controlador → dispositivo
1

¿Qué es la gestión de I/O?

I/O significa Input/Output: entrada y salida de datos entre la computadora y sus dispositivos.

Entrada

Datos que ingresan al sistema desde teclado, mouse, sensores, red, cámara, disco u otros dispositivos.

Salida

Datos que el sistema envía hacia pantalla, impresora, parlantes, archivos, red u otros dispositivos.

Administración

El SO organiza permisos, prioridades, buffers, interrupciones, colas de espera y acceso seguro al hardware.

Mensaje clave

La gestión de I/O evita que cada programa tenga que saber cómo funciona físicamente cada dispositivo. El SO actúa como intermediario.

2

Componentes principales

La comunicación con dispositivos ocurre por capas.

Capas de comunicación

ElementoFunción
Proceso Solicita leer, escribir o comunicarse.
Llamada al sistema Entrada controlada al kernel.
Kernel Coordina recursos, permisos y planificación.
Driver Traduce órdenes generales a órdenes específicas del dispositivo.
Controlador Hardware que maneja directamente el dispositivo.
Dispositivo Elemento físico: disco, teclado, impresora, red, etc.

Flujo típico de una operación

  1. Un programa solicita una operación de entrada/salida.
  2. El SO valida la solicitud mediante una llamada al sistema.
  3. El kernel consulta el driver correspondiente.
  4. El driver se comunica con el controlador del dispositivo.
  5. El dispositivo realiza la operación.
  6. El SO informa el resultado al proceso.

Técnicas usadas por el SO

El SO necesita resolver diferencias de velocidad, disponibilidad y acceso concurrente.

Buffering

Usa una zona temporal de memoria para guardar datos mientras se transfieren entre proceso y dispositivo.

Spooling

Coloca trabajos en una cola para que un dispositivo lento los procese de a uno, como una impresora.

Interrupciones

El dispositivo avisa al CPU cuando necesita atención o cuando terminó una operación.

DMA

Permite transferir datos entre memoria y dispositivo sin que la CPU copie byte por byte.

5

Analogías con la vida real

Para entender I/O, pensemos en situaciones cotidianas.

El SO como recepcionista

En una empresa, los empleados no entran directamente a cada oficina técnica. La recepción recibe pedidos, valida permisos y deriva al área correcta.

Equivalencia: programa → SO → driver → dispositivo.

Buffer como bandeja de cocina

Un mozo deja platos en una bandeja antes de llevarlos a la mesa. La bandeja permite ordenar el flujo y compensar diferencias de velocidad.

Equivalencia: buffer → memoria temporal de intercambio.

Spooling como cola de atención

En una farmacia, los clientes toman número. Aunque lleguen muchos pedidos, se atienden ordenadamente de a uno.

Equivalencia: cola de impresión o cola de trabajos de I/O.

Interrupción como timbre

No hace falta mirar la puerta todo el tiempo. Cuando alguien llega, toca timbre y se atiende el evento.

Equivalencia: interrupción → aviso del dispositivo al CPU.

6

Problemas que debe resolver el SO

Diferencia de velocidades

La CPU es mucho más rápida que muchos dispositivos. Por eso se usan buffers, interrupciones y colas.

Acceso concurrente

Varios procesos pueden querer usar el mismo dispositivo. El SO debe ordenar, bloquear o compartir el acceso.

Seguridad y permisos

No todos los procesos deben poder acceder libremente a discos, cámaras, red o impresoras.

Ejercicio práctico en C++

Simulador de cola de impresión con gestión de I/O.

Consigna

Desarrollar un programa en C++ que simule una cola de impresión administrada por el sistema operativo.

El sistema deberá permitir:

  • Agregar trabajos de impresión.
  • Listar trabajos pendientes.
  • Procesar el siguiente trabajo.
  • Cancelar un trabajo por identificador.
  • Mostrar estadísticas de la cola.
  • Salir del sistema.
Relación con I/O: la impresora es un dispositivo lento, por eso el SO organiza los trabajos mediante una cola de spooling.

Requisitos sugeridos

  1. Crear una estructura TrabajoImpresion con id, nombreArchivo y cantidadPaginas.
  2. Usar una cola o vector para almacenar trabajos pendientes.
  3. Validar que la cantidad de páginas sea mayor a cero.
  4. Procesar los trabajos en orden de llegada.
  5. Informar cuando la cola esté vacía.
  6. Mostrar cantidad de trabajos pendientes y total de páginas en espera.

Código base orientativo

El siguiente código no resuelve todo el ejercicio, pero muestra una estructura similar a la que se puede usar.

#include <iostream>
#include <vector>
#include <string>
using namespace std;

struct TrabajoImpresion {
    int id;
    string nombreArchivo;
    int paginas;
};

void listarTrabajos(const vector<TrabajoImpresion>& cola) {
    if (cola.empty()) {
        cout << "No hay trabajos pendientes.\n";
        return;
    }

    cout << "Trabajos pendientes:\n";
    for (const auto& trabajo : cola) {
        cout << "ID: " << trabajo.id
             << " | Archivo: " << trabajo.nombreArchivo
             << " | Paginas: " << trabajo.paginas << "\n";
    }
}

void procesarSiguiente(vector<TrabajoImpresion>& cola) {
    if (cola.empty()) {
        cout << "La cola de impresion esta vacia.\n";
        return;
    }

    TrabajoImpresion trabajo = cola.front();
    cola.erase(cola.begin());

    cout << "Procesando impresion de: "
         << trabajo.nombreArchivo
         << " (" << trabajo.paginas << " paginas)\n";
}

Guía de resolución del ejercicio

1Modelar el trabajo

Definir qué datos necesita cada trabajo: identificador, nombre del archivo, páginas y eventualmente prioridad.

2Elegir estructura

Usar queue si solo se procesa FIFO, o vector si se desea cancelar trabajos por id.

3Crear menú

Implementar un ciclo que se repita hasta seleccionar Salir.

4Implementar operaciones

Agregar, listar, procesar, cancelar y mostrar estadísticas.

5Validar entradas

Controlar páginas negativas, ids inexistentes, nombres vacíos y cola vacía.

6Relacionar con SO

Explicar en un comentario final cómo el programa representa spooling, cola de I/O y dispositivo lento.
